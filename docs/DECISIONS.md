## Added

- **The Lost Cities + `novus` profile — OFF by default on client, FORCED on server** (2026-06-15, 1.1.1; final) — LC (`lostcities-1.20-7.4.13`, side `both`) ships a custom `novus` profile (`config/lostcities/profiles/novus.json`, `landscapeType: default` → cities overlaid on **unchanged vanilla biomes/terrain**, so vanilla seeds/structure locations still hold). **Client/singleplayer: LC is OFF by default.** The LC world-creation menu is hardcoded to "Disabled" (decompiled — `LostCitySetup.reset()`/`<init>` set the profile field to null; **no config of any tier changes the menu default**), so fresh SP worlds get a vanilla overworld; players opt in by selecting "novus" in that menu. `common.toml` carries no overworld mapping. *(Updated 2026-06-16: `defaultconfigs/lostcities-server.toml`, which had pre-seeded each new world's serverconfig with `selectedProfile = ""`, was removed — SP profile choice is now left entirely to the LC menu, with the mod's own empty spec default applying; off-by-default behavior unchanged.)* **Dedicated server: novus forced on for everyone** via the `_server_deploy` build overlay (below). **Why the split (decompiled `Config.getProfileForDimension`):** the overworld profile resolves from `dimensionsWithProfiles` first, then is overridden by `selectedProfile` only when non-empty — so a `dimensionsWithProfiles` force can't be disabled from the menu, AND the menu can't be made to *default* to novus. Three server-only splits: (1) `config/lostcities/common.toml` forces `minecraft:overworld=novus` (no opt-out); (2) `config/sereneseasons/seasons.toml` week-long season balance (`sub_season_duration=168` + `progress_season_while_offline=true`); (3) `kubejs/server_scripts/default_gamerules.js` sets `playersSleepingPercentage 101` so night-skip can't fire and the season clock stays wall-clock (season + gamerule go together). The prior in-pack `.disabled` copies were removed 2026-06-15; `_server_deploy/` is the sole home.

- **Forge `defaultconfigs/`** ships per-world server defaults — drop `<modid>-server.toml` there.
- **Lost Cities overworld profile — resolution order (decompiled `Config.getProfileForDimension`, 2026-06-15):** the overworld profile is resolved by, in order, (1) loading every `config/lostcities/common.toml` → `dimensionsWithProfiles` entry into the cache, then (2) **if `selectedProfile` is non-empty**, overwriting the overworld entry with it. Consequences: `selectedProfile` (per-world `lostcities-server.toml`, what the world-creation menu sets) **overrides** `dimensionsWithProfiles` when set, but an **empty** `selectedProfile` (menu "Disabled") falls back to whatever `dimensionsWithProfiles` holds. So a `minecraft:overworld=<profile>` force in `dimensionsWithProfiles` **cannot be turned off from the menu**. Also note: the world-creation menu is hardcoded to "Disabled" and reads no config, so it can't be made to default to a profile. **Novus final model:** client ships `selectedProfile = ""` (off by default; players opt into novus in the menu); the dedicated server is forced on via the `_server_deploy` overlay's `config/lostcities/common.toml` (`dimensionsWithProfiles=minecraft:overworld=novus`). Since a force can't be menu-disabled and the menu can't default-on, the client/server split is the only way to get both "vanilla default in SP" and "guaranteed on the server." `defaultconfigs`/serverconfig only seeds at world creation — existing worlds keep their written `selectedProfile`.
- **ModernFix structure cache** is DFU-only, ~80–100 entry ceiling — don't pursue prewarm scripts.
- **KubeJS + CraftTweaker coexist deliberately** — each covers Forge events / mod APIs the other lacks; don't migrate CT→KubeJS.
